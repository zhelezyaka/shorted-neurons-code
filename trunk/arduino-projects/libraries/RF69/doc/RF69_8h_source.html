6 /// \mainpage RF69 library for Arduino
7 ///
8 /// This is the Arduino RF69 library.
9 /// It provides an object-oriented interface for sending and receiving data messages with Hope-RF
10 /// RF69B based radio modules, such as the RFM69 module, as used on Moteino and Moteino-USB
11 /// http://lowpowerlab.com/moteino/
12 /// and compatible chips and modules such as RFM69W, RFM69HW, RFM69CW, RFM69HCW (Semtech SX1231, SX1231H)
13 /// The RFM69 device is described in http://www.hoperf.cn/upload/rf/RFM69-V1.3.pdf
14 /// and http://www.hoperf.com/upload/rfchip/RF69-V1.2.pdf
15 ///
16 /// The Hope-RF (http://www.hoperf.com) RF69 is a low-cost ISM transceiver
17 /// chip. It supports FSK, GFSK, OOK over a wide range of frequencies and
18 /// programmable data rates. It also suports AES encryption of up to 64 octets
19 /// of payload It is available prepackaged on modules such as the RFM69W. And
20 /// such modules can be prepacked on processor boards such as the Moteino from
21 /// LowPowerLabs (which is what we used to develop the RF69 library)
22 ///
23 /// This library provides functions for sending and receiving messages of up
24 /// to 60 octets on any frequency supported by the RF69, in a range of
25 /// predefined data rates and frequency deviations. Frequency can be set with
26 /// 61Hz precision to any frequency from 240.0MHz to 960.0MHz.
27 ///
28 /// Up to 2 RF69B modules can be connected to an Arduino (3 on a Mega),
29 /// permitting the construction of translators and frequency changers, etc.
30 ///
31 /// This library provides classes for
32 /// - RF69: unaddressed, unreliable messages
33 ///
34 /// Coming soon: classes for datagrams, reliable datagrams, routers and meshes
35 ///
36 /// The following modulation types are suppported with a range of modem configurations for
37 /// common data rates and frequency deviations:
38 /// - GFSK Gaussian Frequency Shift Keying
39 /// - FSK Frequency Shift Keying
40 ///
41 /// Support for other RF69 features such as on-chip temperature measurement,
42 /// transmitter power control etc is also provided.
43 ///
44 /// The latest version of this documentation can be downloaded from
45 /// http://www.airspayce.com/mikem/arduino/RF69
46 ///
47 /// Example Arduino programs are included to show the main modes of use.
48 ///
49 /// The version of the package that this documentation refers to can be downloaded
50 /// from http://www.airspayce.com/mikem/arduino/RF69/RF69-1.0.zip
51 /// You can find the latest version at http://www.airspayce.com/mikem/arduino/RF69
52 ///
53 /// Tested on USB-Moteino with arduino-1.0.5
54 /// on OpenSuSE 13.1
55 ///
56 /// \par Packet Format
57 ///
58 /// All messages sent and received by this RF22 library must conform to this packet format:
59 ///
60 /// - 4 octets PREAMBLE
61 /// - 2 octets SYNC 0x2d, 0xd4 (configurable, so you can use this as a network filter)
62 /// - 1 octet RF69 payload length
63 /// - 4 octets HEADER: (TO, FROM, ID, FLAGS)
64 /// - 0 to 60 octets DATA
65 /// - 2 octets CRC computed with CRC16(IBM), computed on HEADER and DATA
66 ///
67 /// For technical reasons, the message format is not compatible with the
68 /// 'HopeRF Radio Transceiver Message Library for Arduino'
69 /// http://www.airspayce.com/mikem/arduino/HopeRF from the same author. Nor is
70 /// it compatible with messages sent by 'Virtual Wire'
71 /// http://www.airspayce.com/mikem/arduino/VirtualWire.pdf also from the same
72 /// author. Nor is it compatible with messages sent by 'RF22'
73 /// http://www.airspayce.com/mikem/arduino/RF22 also from the same author.
74 ///
75 /// \par Connecting RFM-22 to Arduino
76 ///
77 /// We tested with Moteino, which is an Arduino Uno compatible with the RFM69W
78 /// module on-board. Therefore it needs no connections other than the USB
79 /// programming connection and an antenna to make it work.
80 ///
81 /// If you have a bare RFM69W that you want to connect to an Arduino, you
82 /// might use these connections (untested): CAUTION: you must use a 3.3V type
83 /// Arduino, otherwise you will also need voltage level shifters between the
84 /// Arduino and the RFM69. CAUTION, you must also ensure you connect an
85 /// antenna
86 ///
87 /// \code
88 /// Arduino RFM69W
89 /// GND----------GND (ground in)
90 /// 3V3----------3.3V (3.3V in)
91 /// interrupt 0 pin D2-----------DIO0 (interrupt request out)
92 /// SS pin D10----------NSS (chip select in)
93 /// SCK pin D13----------SCK (SPI clock in)
94 /// MOSI pin D11----------MOSI (SPI Data in)
95 /// MISO pin D12----------MISO (SPI Data out)
96 /// \endcode
97 ///
98 /// With these connections, you can then use the default constructor RF69().
99 /// You can override the default settings for the SS pin and the interrupt in
100 /// the RF69 constructor if you wish to connect the slave select SS to other
101 /// than the normal one for your Arduino (D10 for Diecimila, Uno etc and D53
102 /// for Mega) or the interrupt request to other than pin D2 (Caution,
103 /// different processors have different constraints as to the pins available
104 /// for interrupts).
105 ///
106 /// It is possible to have 2 or more radios connected to one Arduino, provided
107 /// each radio has its own SS and interrupt line (SCK, SDI and SDO are common
108 /// to all radios)
109 ///
110 /// Caution: on some Arduinos such as the Mega 2560, if you set the slave
111 /// select pin to be other than the usual SS pin (D53 on Mega 2560), you may
112 /// need to set the usual SS pin to be an output to force the Arduino into SPI
113 /// master mode.
114 ///
115 /// Caution: Power supply requirements of the RF69 module may be relevant in some circumstances:
116 /// RF69 modules are capable of pulling 45mA+ at full power, where Arduino's 3.3V line can
117 /// give 50mA. You may need to make provision for alternate power supply for
118 /// the RF69, especially if you wish to use full transmit power, and/or you have
119 /// other shields demanding power. Inadequate power for the RF69 is likely to cause symptoms such as:
120 /// -reset's/bootups terminate with "init failed" messages
121 /// -random termination of communication after 5-30 packets sent/received
122 /// -"fake ok" state, where initialization passes fluently, but communication doesn't happen
123 /// -shields hang Arduino boards, especially during the flashing
124 /// \par Interrupts
125 ///
126 /// The RF69 library uses interrupts to react to events in the RF69 module,
127 /// such as the reception of a new packet, or the completion of transmission
128 /// of a packet. The RF69 library interrupt service routine reads status from
129 /// and writes data to the the RF69 module via the SPI interface. It is very
130 /// important therefore, that if you are using the RF69 library with another
131 /// SPI based deviced, that you disable interrupts while you transfer data to
132 /// and from that other device. Use cli() to disable interrupts and sei() to
133 /// reenable them.
134 ///
135 /// \par Memory
136 ///
137 /// The RF69 library requires non-trivial amounts of memory. The sample
138 /// programs above all compile to about 8kbytes each, which will fit in the
139 /// flash proram memory of most Arduinos. However, the RAM requirements are
140 /// more critical. Therefore, you should be vary sparing with RAM use in
141 /// programs that use the RF69 library.
142 ///
143 /// It is often hard to accurately identify when you are hitting RAM limits on Arduino.
144 /// The symptoms can include:
145 /// - Mysterious crashes and restarts
146 /// - Changes in behaviour when seemingly unrelated changes are made (such as adding print() statements)
147 /// - Hanging
148 /// - Output from Serial.print() not appearing
149 ///
150 /// \par Automatic Frequency Control (AFC)
151 ///
152 /// The RF69 module is configured by the RF69 library to always use AFC.
153 ///
154 /// \par Performance
155 ///
156 /// Some simple speed performance tests have been conducted.
157 /// In general packet transmission rate will be limited by the modulation scheme.
158 /// Also, if your code does any slow operations like Serial printing it will also limit performance.
159 /// We disabled any printing in the tests below.
160 /// We tested with RF69::GFSK_Rb250Fd250, which is probably the fastest scheme available.
161 /// We tested with a 13 octet message length, over a very short distance of 10cm.
162 ///
163 /// Transmission (no reply) tests with modulation RF69::GFSK_Rb250Fd250 and a
164 /// 13 octet message show about 152 messages per second transmitted and received.
165 ///
166 /// Transmit-and-wait-for-a-reply tests with modulation RF69::GFSK_Rb250Fd250 and a
167 /// 13 octet message (send and receive) show about 68 round trips per second.
168 ///
